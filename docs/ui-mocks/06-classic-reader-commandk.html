Live local workspace viewer

Pathlens opens a local directory as a live browser-based workspace. The file tree remains visible, several files can stay open as tabs, and the active viewer updates without a full page refresh.

Primary workflow

Run the CLI in a directory, open the generated local URL, and browse Markdown, HTML, code, text, and image files from one interface.

  • The sidebar is the stable spatial map of the directory.
  • Tabs preserve working context across Markdown, HTML, and source files.
  • The right outline makes long Markdown documents navigable.
  • Command K is the fast path for keyboard-heavy file switching.

Live update model

The backend watches filesystem events and streams semantic updates to the SPA. File additions and removals update the tree. Changes to the active file refresh only the viewer body. 

# start the viewer
pathlens . --open

# preview generated artifacts
pathlens ./dist --include md,html,ts,tsx,json,png

Viewer strategy

Markdown renders as a readable document. HTML opens in a sandboxed preview with a source toggle. Code files use syntax highlighting. Large or unsupported files degrade to a safe raw/download view.
