Vivi review concept

Right inspector draft absorption

Three Stage 1 directions for retiring the separate Comments hub as a primary destination. Cmd/Ctrl+Shift+C becomes a focus shortcut into the inspector, while private drafts, open threads, and reviewed history stay in one review navigation surface.

A Section Stack B Review Lanes C Draft Shelf
Concept A

Section Stack

Keep the current Review inspector structure and add Drafts as a first-class section between In Review and In This File.

Smallest change Current model Good default

Best when the user mostly reviews one active file at a time and only needs a clear private-draft checkpoint before publishing.

Concept B

Review Lanes

Treat the inspector as a state switcher: Attention, Drafts, Open, and Reviewed are equal lanes under the same Review header.

Best triage Lifecycle explicit Dense sessions

Best when review sessions have many threads and drafts across multiple files. The tradeoff is that the right side becomes more inbox-like and less document-inspector-like.

Concept C

Draft Shelf

Keep Review Queue and In This File calm, but reserve a sticky draft shelf inside the inspector whenever private drafts exist.

Least disruptive Persistent CTA Reader friendly

Best when drafts are important but should not compete with active reading. The tradeoff is weaker cross-file triage unless the shelf expands into a full list.
