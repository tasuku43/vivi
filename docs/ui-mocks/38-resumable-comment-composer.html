Vivi concept 38 · comment interruption policy

Keep the reading flow; never lose typed feedback

This explores what happens when a reviewer clicks another line, file, tab, viewer mode, inspector item, or blank area while composing. It keeps Vivi a workspace viewer—not a Crit-style stacked review page.

Publish is the agent boundary No resident inbox Tree + tabs stay primary
Reviewer action A · Resumable inline B · Single focus C · Draft shelf
Click another line Keep current composer; open another inline. Collapse current into a resumable chip; open one editor. Autosave current into inspector; open next editor.
Click blank content Do nothing. Outside click never discards. Collapse to chip. Move to shelf.
Switch tab/file/mode Retain typed body locally by file + anchor; restore on return. Retain one collapsed chip per file. Draft remains globally visible in inspector.
Escape / × Empty closes. Typed collapses first; explicit Discard removes. Always collapses; chip menu owns Discard. Closes editor because shelf already holds text.
File changes underneath Keep text, mark anchor stale, offer Re-anchor / Discard. Same, on the chip. Same, in the shelf card.
Save → Publish Save creates a private pending draft. Publish exposes saved drafts to agents. Agent fetch timing remains independent.
Recommended · least surprising

A · Resumable inline

Inline composers stay where the reviewer made them. Navigating away never publishes and never silently loses typed text.

  • Best fit for Vivi's spatial file-viewer model.
  • Multiple same-file thoughts can coexist.
  • Needs a small local “unsent” state separate from saved drafts.

B · Single focus

Only one full editor is open. Clicking elsewhere collapses typed text into an anchor chip rather than closing it.

  • Low visual noise for long documents.
  • More hidden state; reviewers must notice chips.
  • Good fallback for narrow layouts.

C · Draft shelf

Typing is continuously retained in a right-inspector shelf. The editor may close freely because its text remains visible globally.

  • Strongest cross-file recovery.
  • Competes with outline, metadata, and Review Queue.
  • Feels closer to task management than file viewing.
