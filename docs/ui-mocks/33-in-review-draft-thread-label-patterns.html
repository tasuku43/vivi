Vivi review concept

In Review draft thread labels

Four ways to show private draft threads inside the existing hairline In Review file expansion. Each pattern keeps drafts in the same review state as open threads, while making agent visibility unmistakable without turning rows into cards.

A Mixed Count B Status Column C Grouped Threads D Draft Pending
Pattern A

Mixed Count Badges

The file row keeps the current compact shape but splits the count: published threads and pending threads are separate pills.

Closest to current UI Strong file-level summary

Best if the file row must stay dense. The tradeoff is that users read the exact thread state only after expanding the row.

Pattern B

Status Column

Thread rows get a fixed left status column. Open, Attention, and Draft are read before the location or preview text.

Fastest row scan Best mixed lists

Best when expanded files can have several mixed thread states. The tradeoff is a slightly wider thread row.

Pattern D

Draft Pending Label

Keep every row named "thread", but draft rows use a stronger compound status: Draft pending. The label says what happens next.

Most explicit Good first-run clarity

Best if "draft thread" might be confusing. The tradeoff is longer labels and slightly more visual weight.
