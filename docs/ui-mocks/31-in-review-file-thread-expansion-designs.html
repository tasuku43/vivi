In Review File Expansion Designs

These variants keep the existing right inspector structure intact: Queued, In Review, and Reviewed remain the same sections. Only the file rows inside In Review can expand, and the expanded area lists comment threads as shallow rows with source-jump actions.

Hairline Indented Table Preview Strong Minimal

Design A

Hairline Threads

The expanded area stays almost flush with the existing In Review list. Threads are separated by hairlines, not cards or nested containers.

Quietest Best fit Low emphasis
Experience
The user expands a file and gets a compact thread list without feeling like the inspector changed modes.
Best for
Preserving the current sidebar rhythm while adding just enough comment access.
Tradeoff
Thread rows are subtle, so source-jump affordances must stay clear.

Design B

Indented Threads

Threads are lightly indented under the expanded file. This makes ownership obvious while still avoiding a card-in-card feeling.

Clear ownership A bit more structure Still shallow
Experience
The file row remains the parent object. Threads read as direct children of that file, not as a separate comments mode.
Best for
A design that needs a clearer “these threads belong to this file” signal.
Tradeoff
Slightly busier than Hairline Threads because indentation creates more visual geometry.

Design C

Compact Table Rows

Threads use a small table-like rhythm: anchor, preview, and action. It is denser and more systematic.

Most compact Scannable anchors More utilitarian
Experience
A file can expose several threads without consuming too much vertical space.
Best for
Dense review sessions where anchor labels matter more than comment prose.
Tradeoff
It feels more like an operational list and less like a reader companion.
