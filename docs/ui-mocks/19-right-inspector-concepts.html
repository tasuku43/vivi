Right inspector after concepts

Vivi is a local review adapter: humans read visual artifacts, then leave feedback that agents can consume through stable contracts. These mocks keep the right inspector as review navigation first, while making its secondary reading aids adapt to the active context.

Review target lifecycle

A changed file is not always active review work. The right inspector should show files while the user still needs to decide, respond, or verify. Accepted files and fully resolved conversations should leave the active queue.

Detected HEAD diff or watcher signal creates a review candidate. visible
Seen User opened it; no decision has been recorded yet. visible
Accepted User reviewed and accepts the change as-is. hidden
In review User commented; the file now represents an active feedback loop. visible
Resolved All threads are resolved after agent replies and fixes. hidden

Two axes, one queue

Read state is about attention. Review state is about lifecycle. The inspector should prioritize unread active work, but hide completed work even if it was recently active.

Unread + reviewing Agent replied, new thread, or status changed. Highest priority. show first
Read + reviewing Still has open threads or no explicit accept decision. show
Unread + candidate New HEAD change not opened yet. Needs first-pass review. show
Read + complete Accepted as-is, resolved, or archived. Available in history only. hide
